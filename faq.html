FAQ

Q1. What is ImageWarp?

A1. ImageWarp is a high-performance image analysis and editing software application, which combines the power of a graphic development environment with a comprehensive image analysis toolset. ImageWarp can be used with numerous acquisition devices to provide real-time image acquisition or simply as a tool for analyzing static images. Download the 2-page brochure by clicking here [1.18MB].

Q2. How does ImageWarp fit in with the rest of BitFlow's imaging products?

A2. ImageWarp fills a void in BitFlow's current product line by offering a well-integrated software solution that fully supports our existing line of frame grabber products. You can now use BitFlow's products for more than camera interfacing and acquisition. By taking advantage of ImageWarp's sophisticated BitFlow driver and our superior FlowThru architecture, we can now offer you a complete solution.

Q3. What Image Analysis/Image Processing functionality does ImageWarp have?

A3. ImageWarp contains hundreds of functions for performing image analysis and processing. Refer to Features for a complete list of ImageWarp functions.

Q5. What acquisition/imaging devices does ImageWarp currently support?

A5. ImageWarp fully supports the entire line of BitFlow frame grabbers as well as standard imaging devices such as Fire Wire, USB cameras (via VFW and WDM drivers) and TWAIN devices. Support for other frame grabber products will be evaluated on a case-by-case basis, with the decision ultimately determined by the size of the opportunity.

NOTE: In order to use BitFlow products with ImageWarp, you will need to have the FREE drivers-only version of SDK 4.00 or higher installed on your computer.

Q6. What programming environment can I use to write applications using ImageWarp?

A6. ImageWarp features a VB-like scripting tool in order to develop applications for use within the ImageWarp environment. An SDK that allows direct access to the highly optimized algorithms is in development and will be made available soon. There are currently more than 30 example applications that can be used to assist you with the creation of your own application. In addition, ImageWarp offers an extensive set of DDE functions which allows users to control ImageWarp from a third party application and exchange images and data. Using the DDE mechanism, ImageWarp can be instructed to run in an invisible mode, so that the end-user can be presented with a custom user interface. A console-based example of this capability is provided in the documentation.

Q7. Can ImageWarp be interfaced to external devices such as PLC?

A7. Yes. ImageWarp has a comprehensive set of serial communication and I/O functions that can be used to interchange data and signals with other computers, machines, and laboratory equipment. System integrators can use ImageWarp to build fully automated vision systems for quality control, web inspection, robotic assembly, and many other applications.

Q8. Can multiple copies of ImageWarp run simultaneously on a single PC?

A8. Yes. In fact, by creating several copies of ImageWarp's root directory along with the driver folder, you can also run each copy with a separate video source. In addition, we are currently adding the ability for each copy of ImageWarp to communicate with each other in the event synchronization is required (see previous question).

Q9. Can ImageWarp be customized to fit my requirements?

A9. ImageWarp includes numerous example scripts and images designed to demonstrate its capabilities. You can also provide us with a sample images in order to perform a 'test' analysis.

In the event that you require ImageWarp to be customized to suit your application requirements, BitFlow will be more than happy to evaluate these requirements and provide a quotation or recommend a 3rd Party for these services.

Q10. Is a demo version of ImageWarp available?

A10. Yes. ImageWarp has 2 demonstration levels that can either be downloaded from Download page or installed from a demo CD. In order to use the demos, you will need to have a network card present in your computer.

Each demo level has certain limitations relative to the licensed commercial release.

Limited Demo:

  • 14-day trial period
  • Only the Simulation Device is supported (no support for BitFlow frame grabbers, Fire Wire, VFW, WDM or TWAIN devices)
  • Only demo images can be opened from the hard drive
  • Images can be saved in the ImageWarp proprietary format (IWD) only
  • Images cannot be copied to or pasted from the Clipboard
  • A number of selected functions are disabled
  • Execution of scripts is interrupted every 60 seconds

The Limited Demo can be upgraded to the Registered Demo through the registration procedure. The Registered Demo has full functionality of the commercial release with the following exceptions:

  • 21-day trial period
  • Only BitFlow and Fire Wire (IEEE-1394) camera interfaces are supported for video capture
  • Images can be saved in the ImageWarp proprietary format (IWD) only
  • Images cannot be copied to or pasted from the Clipboard
  • Only a limited amount of measurement data can be recorded or exported
